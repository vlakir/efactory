# `ports/` — Protocol-интерфейсы

Границы шестиугольника. `typing.Protocol` для структурной типизации
(а не `ABC`) — fake-порты в тестах не требуют наследования.

## Структура

- `inbound/` — что система **предлагает** миру: CLI-команды,
  MCP-tools, HTTP-handlers (когда появятся), GUI-actions.
- `outbound/` — что система **требует** от мира: репозитории,
  AI-провайдер, файловое хранилище, графовая БД, MCP-клиент.

## Сюда

- Только `Protocol`-классы с сигнатурами методов и docstring-ами
  «зачем нужно, что делает».
- Доменные типы в сигнатурах (модели из `domain/`, value objects).

## НЕ сюда

- Реализации (они в `adapters/`).
- `sqlalchemy`, `kuzu`, `anthropic`, `mcp`, `typer` импорты —
  технологические детали за портами.
- Импорты из `application/`, `adapters/`, `composition/`.

## Зависимости

`domain/` и stdlib (включая `typing.Protocol`). Всё.

## Outbound-порты на старте

- `MetadataRepository` — CRUD метаданных проектов и сущностей.
- `TopologyGraphStore` — графовые узлы и рёбра топологий.
- `ProjectFileRepository` — чтение/запись файлов проекта.
- `AIProvider` — отправка сообщений в LLM с tool use.
- `MCPClient` — клиент внешних MCP-серверов.

## Inbound-порты на старте

- `CLI` — командная строка.
- `MCPServer` — наш MCP-сервер для внешних ИИ.

Полная реализация только тех портов, что нужны Walking Skeleton.
Остальные — Protocol-каркасы с TODO; их реальная реализация — в
задачах дорожной карты (T004 / T010 / T012 / T013 / ...).
